# Уточнение: Запись на пробное занятие

## 1. Граничные случаи (Edge Cases)

### EC-1: Гонка за последнее место
**Сценарий:** Два родителя одновременно записывают детей на последнее место.
**Решение:** SERIALIZABLE транзакция в PostgreSQL. Проигравший получает 409 Conflict с сообщением "NO_SEATS". Frontend перезапрашивает статус мест и обновляет UI.

### EC-2: Возрастная граница
**Сценарий:** Ребенку 5 лет 11 месяцев, класс для 6-10 лет, секция через 2 недели.
**Решение:** Возраст рассчитывается на дату секции (`section.startTime`), а не на дату записи. Если на дату секции ребенку уже 6 — запись разрешена.

### EC-3: Повторная запись после отмены
**Сценарий:** Родитель отменил пробное и хочет записаться снова.
**Решение:** Уникальный индекс `childId + classId + type=TRIAL` включает все статусы. Отмененная запись остается в базе → повторная запись невозможна. Это намеренное бизнес-правило: пробное занятие — одноразовое.

### EC-4: Запись на отмененную/прошедшую секцию
**Сценарий:** Секция была отменена учителем или дата уже прошла.
**Решение:** Проверка `section.status = ACTIVE AND section.startTime > NOW()` в eligibility check.

### EC-5: Ребенок удален из профиля
**Сценарий:** Родитель удалил профиль ребенка, но enrollment существует.
**Решение:** Soft delete для детей. Enrollment привязан к childId, child остается в базе с `deletedAt`.

## 2. Пути ошибок (Error Paths)

| Ошибка | HTTP | Код | Действие frontend |
|--------|------|-----|-------------------|
| Ребенок не принадлежит родителю | 403 | FORBIDDEN | Показать ошибку доступа |
| Класс/секция не найдены | 404 | NOT_FOUND | Перенаправить на каталог |
| Пробное уже использовано | 409 | TRIAL_ALREADY_USED | Показать "Уже записаны", предложить оплату |
| Нет свободных мест | 409 | NO_SEATS | Показать другие секции |
| Возраст не подходит | 422 | AGE_MISMATCH | Показать AgeCheck компонент |
| Секция неактивна | 422 | SECTION_INACTIVE | Скрыть кнопку записи |
| DB connection timeout | 500 | INTERNAL | Retry с exponential backoff |

## 3. Безопасность

| Угроза | Защита |
|--------|--------|
| Массовая запись ботом | Rate limit: 5 записей/минуту на пользователя |
| Запись чужого ребенка | EnrollmentGuard проверяет parentId → childId ownership |
| Перебор sectionId | UUID v4, нет sequential IDs |
| IDOR на DELETE | Проверка: enrollment.child.parentId === currentUser.id |

## 4. Производительность

- Индекс на `(sectionId, status)` для быстрого подсчета мест
- Индекс на `(childId, classId, type)` — уникальный, одновременно для проверки и constraint
- Кэширование `availableSeats` в Redis с инвалидацией при создании/отмене записи (TTL 30s как fallback)

## 5. Доступность (Accessibility)

- TrialBookingCard: `aria-label` для кнопки записи с именем ребенка и класса
- SeatAvailability: `aria-live="polite"` для обновления счетчика
- AgeCheck: `role="alert"` для предупреждения о возрасте
