# Спецификация: Запись на пробное занятие

**Feature ID:** trial-booking
**User Stories:** US-P05
**Epic:** E3
**Приоритет:** Высокий (конверсионная воронка)

---

## 1. Функциональные требования

### FR-1: Запись ребенка на пробное занятие
Родитель может записать ребенка на бесплатное пробное занятие одним нажатием.
Банковская карта не требуется.

### FR-2: Правило одного пробного
Один ребенок может записаться на пробное занятие конкретного класса только один раз.
Повторная запись на тот же класс запрещена (уникальный индекс `childId + classId + type=TRIAL`).

### FR-3: Проверка возраста
Возраст ребенка должен попадать в диапазон `ageMin..ageMax` класса.
Расчет возраста — на дату секции, а не на дату записи.

### FR-4: Проверка свободных мест
Количество активных записей на секцию не должно превышать `maxStudents`.

### FR-5: Отмена записи
Родитель может отменить пробную запись. После отмены повторная запись на тот же класс невозможна.

## 2. Acceptance Criteria (Gherkin)

```gherkin
Feature: Запись на пробное занятие

  Scenario: Успешная запись на пробное
    Given родитель авторизован и находится на карточке класса
    And у класса есть пробная секция со свободными местами
    And возраст ребенка соответствует диапазону класса
    And ребенок еще не записывался на пробное в этот класс
    When родитель нажимает "Записаться на пробное" и выбирает ребенка
    Then создается enrollment с type=TRIAL и status=confirmed
    And количество свободных мест уменьшается на 1
    And родитель получает email-уведомление с деталями

  Scenario: Отказ — ребенок уже был на пробном
    Given ребенок уже имеет enrollment type=TRIAL для данного класса
    When родитель пытается записать этого ребенка на пробное
    Then система показывает ошибку "Пробное занятие для этого класса уже использовано"
    And кнопка записи неактивна

  Scenario: Отказ — возраст не соответствует
    Given возраст ребенка не входит в диапазон ageMin..ageMax класса
    When родитель пытается записать ребенка
    Then система показывает предупреждение о возрастном несоответствии
    And запись не создается

  Scenario: Отказ — нет свободных мест
    Given все места в секции заняты (enrollments.count >= maxStudents)
    When родитель пытается записать ребенка
    Then система показывает "Нет свободных мест"
    And предлагает другие секции этого класса

  Scenario: Отмена пробной записи
    Given родитель имеет активную пробную запись
    When родитель нажимает "Отменить запись"
    Then enrollment переходит в status=cancelled
    And свободное место в секции увеличивается на 1
```

## 3. Нефункциональные требования

| NFR | Требование |
|-----|-----------|
| Производительность | Создание записи < 500ms |
| Консистентность | Атомарная проверка + создание записи (транзакция) |
| Доступность | Endpoint доступен 99.9% времени |
| Безопасность | Только авторизованный родитель может записывать своих детей |

---

## 4. Ограничения и допущения

- Пробное занятие всегда бесплатно (price = 0)
- Календарное событие добавляется через email-приглашение (ICS)
- Push-уведомления реализуются в отдельной фиче notifications
