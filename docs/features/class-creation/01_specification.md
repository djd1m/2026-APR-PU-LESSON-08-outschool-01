# Спецификация: Создание и модерация занятий

## Пользовательские истории

### US-T02: Создание занятия учителем

**Как** учитель,
**я хочу** создать занятие с описанием, расписанием, ценой и предметом,
**чтобы** оно появилось в каталоге после модерации.

**Приоритет:** Critical | **Effort:** L

### US-A01: Модерация занятий администратором

**Как** администратор,
**я хочу** проверять и одобрять/отклонять занятия,
**чтобы** гарантировать качество контента на платформе.

**Приоритет:** Critical | **Effort:** M

## Критерии приемки (Gherkin)

```gherkin
Feature: Создание занятия учителем

  Background:
    Given учитель авторизован

  Scenario: Создание занятия в черновике
    When учитель заполняет форму: название, описание, предмет, возраст (6-10), цена (800), формат (онлайн)
    And нажимает "Сохранить черновик"
    Then занятие создано в статусе DRAFT
    And видно в личном кабинете учителя

  Scenario: Отправка на модерацию
    Given занятие в статусе DRAFT с заполненными обязательными полями
    When учитель нажимает "Отправить на модерацию"
    Then статус меняется на PENDING_REVIEW
    And занятие недоступно для редактирования (кроме отмены)
    And администратор получает уведомление

  Scenario: Отправка неполного занятия на модерацию
    Given занятие без описания (обязательное поле)
    When учитель нажимает "Отправить на модерацию"
    Then отображается ошибка "Заполните описание"
    And статус остается DRAFT

  Scenario: Генерация URL-slug
    When учитель создает занятие с названием "Математика для дошкольников"
    Then генерируется slug "matematika-dlya-doshkolnikov"
    And занятие доступно по /classes/matematika-dlya-doshkolnikov

  Scenario: Дублирующийся slug
    Given занятие со slug "matematika-dlya-doshkolnikov" уже существует
    When создается новое занятие с тем же названием
    Then slug = "matematika-dlya-doshkolnikov-2"

Feature: Модерация занятий администратором

  Background:
    Given администратор авторизован

  Scenario: Одобрение занятия
    Given занятие в статусе PENDING_REVIEW
    When администратор нажимает "Одобрить"
    Then статус меняется на PUBLISHED
    And занятие индексируется в Elasticsearch
    And появляется в публичном каталоге
    And учитель получает уведомление "Занятие одобрено"

  Scenario: Отклонение занятия с причиной
    Given занятие в статусе PENDING_REVIEW
    When администратор вводит причину "Описание не соответствует стандартам"
    And нажимает "Отклонить"
    Then статус меняется на REJECTED
    And учитель видит причину отклонения
    And учитель может исправить и отправить повторно

  Scenario: Отклонение без причины
    Given занятие в статусе PENDING_REVIEW
    When администратор нажимает "Отклонить" без указания причины
    Then отображается ошибка "Укажите причину отклонения"

  Scenario: Повторная отправка после отклонения
    Given занятие в статусе REJECTED
    When учитель редактирует занятие и нажимает "Отправить повторно"
    Then статус меняется на PENDING_REVIEW
    And занятие снова в очереди модерации

  Scenario: Снятие с публикации
    Given занятие в статусе PUBLISHED
    When учитель нажимает "Снять с публикации"
    Then статус меняется на DRAFT
    And занятие удаляется из Elasticsearch индекса
```

## Нефункциональные требования

- Slug: транслитерация кириллицы, lowercase, дефисы вместо пробелов
- Обязательные поля для модерации: title, description, subject, ageMin, ageMax, price, format
- Время модерации: SLA 24 часа (мониторинг backlog)
- Максимум 50 активных занятий на учителя
