# Уточнение: Создание и модерация занятий

## Граничные случаи

### 1. Конкурентные редактирования

| Ситуация | Поведение |
|----------|-----------|
| Учитель редактирует на двух вкладках одновременно | Последнее сохранение побеждает (last-write-wins) |
| Учитель редактирует, админ одобряет | Оптимистичная блокировка: PUT вернет 409 если status изменился |
| Два админа обрабатывают одно занятие | Первый успешен, второй получит 409 "Статус уже изменен" |

**Реализация:** поле `version` (auto-increment) в модели Class. PUT и status transitions проверяют `WHERE version = expected_version`.

### 2. Модерационный backlog

| Ситуация | Поведение |
|----------|-----------|
| Более 50 занятий в PENDING_REVIEW | Алерт администраторам |
| Занятие в PENDING > 24 часов | Эскалация (email старшему администратору) |
| Занятие в PENDING > 72 часов | Автоматическое уведомление учителю "Ваше занятие в очереди" |
| Нет администраторов онлайн | Backlog растет, SLA нарушается — мониторинг |

### 3. UX отклонения

| Ситуация | Поведение |
|----------|-----------|
| Учитель видит REJECTED | Отображается причина + кнопка "Исправить и отправить повторно" |
| Причина отклонения < 10 символов | Backend: 400 "Причина должна быть не менее 10 символов" |
| Учитель исправил и отправил повторно | REJECTED → PENDING_REVIEW, причина очищается |
| Учитель нажал "Отправить повторно" без изменений | Допускается — админ может снова отклонить |

### 4. Генерация slug

| Ввод | Ожидаемый slug |
|------|---------------|
| "Математика для дошкольников" | `matematika-dlya-doshkolnikov` |
| "English for Kids!" | `english-for-kids` |
| "  Пробелы  по краям  " | `probely-po-krayam` |
| "123 Урок" | `123-urok` |
| "" (пустое название) | Невозможно — title обязателен |
| Название 300 символов | Slug обрезается до 100 символов |
| Дубликат slug | Добавляется суффикс: `-2`, `-3` |

### 5. Лимиты

| Параметр | Значение | При превышении |
|----------|----------|----------------|
| Активные занятия на учителя | 50 | 400 "Достигнут лимит занятий" |
| Длина title | 200 символов | 400 "Название не может быть длиннее 200 символов" |
| Длина description | 5000 символов | 400 |
| Цена max | 50 000 руб | 400 |
| Цена min | 0 (бесплатные занятия допускаются) | - |
| ageMin | 3 | 400 |
| ageMax | 18 | 400 |

## Пути ошибок

### Невалидные переходы статусов

| Текущий статус | Попытка | Результат |
|----------------|---------|-----------|
| DRAFT | approve | 400 "Переход из DRAFT в PUBLISHED невозможен" |
| PUBLISHED | submit | 400 "Переход из PUBLISHED в PENDING_REVIEW невозможен" |
| REJECTED | approve | 400 (нужно сначала resubmit) |
| PENDING_REVIEW | edit (PUT) | 400 "Нельзя редактировать занятие на модерации" |

### Ошибки индексации

| Ситуация | Поведение |
|----------|-----------|
| ES недоступен при approve | Статус меняется на PUBLISHED, индексация ставится в retry queue |
| Retry queue переполнен | Алерт, ручная реиндексация через CLI |
| Занятие в PostgreSQL, но не в ES | Cron-задача реиндексации обнаружит рассинхрон |

## Безопасность

- POST/PUT/DELETE /classes — только авторизованный учитель (JwtAuthGuard + TeacherRole)
- ClassOwnerGuard — учитель может редактировать/удалять только свои занятия
- approve/reject — только @Roles(Role.ADMIN)
- Описание и название: санитизация HTML, защита от XSS
- Slug: только [a-z0-9-], невозможна инъекция через URL
- Удаление: soft delete (не физическое), для возможности аудита

## Конкурентность и целостность

- Optimistic locking через version field на Class
- Транзакция при approve: update status + index ES (если ES fail — rollback)
- При resubmit: rejectionReason очищается в той же транзакции
