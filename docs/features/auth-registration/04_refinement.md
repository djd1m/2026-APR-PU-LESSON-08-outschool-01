# Уточнение: Аутентификация и регистрация

## Граничные случаи

### 1. Истечение токенов

| Ситуация | Поведение |
|----------|-----------|
| Access-токен истек, refresh валиден | Клиент вызывает POST /auth/refresh, получает новую пару |
| Оба токена истекли | Редирект на страницу входа, сессия потеряна |
| Refresh-токен отозван (revoked) | 401, требуется повторный вход |
| Параллельные запросы refresh | Только первый успешен (ротация), остальные получают 401 |

### 2. Ротация refresh-токенов

При каждом обновлении старый refresh-токен отзывается. Если злоумышленник перехватит старый токен и попытается использовать его после ротации — система обнаружит повторное использование отозванного токена и может инвалидировать все refresh-токены пользователя (цепочка отзыва).

### 3. OAuth-ошибки

| Ситуация | Поведение |
|----------|-----------|
| VK/Яндекс недоступен | Показать пользователю ошибку, предложить вход по email |
| OAuth callback с ошибкой (user_denied) | Редирект на /login с сообщением "Вход отменен" |
| Email из OAuth уже занят LOCAL аккаунтом | Привязка OAuth к существующему аккаунту (merge) |
| OAuth без email (VK может не возвращать) | Создание аккаунта без email, предложить указать позже |

### 4. Brute force защита

- 5 неудачных попыток за 15 минут — временная блокировка аккаунта
- Блокировка по IP через rate limiter (nestjs/throttler): 5 req/min на /auth/login
- После блокировки аккаунта: автоматическая разблокировка через 15 минут
- Администратор может разблокировать вручную

### 5. Невалидные входные данные

| Поле | Валидация | Ошибка |
|------|-----------|--------|
| email | RFC 5322, max 255 символов | 400 "Некорректный email" |
| password | min 8, upper 1, digit 1, special 1 | 400 "Пароль не соответствует требованиям" |
| role | enum [PARENT, TEACHER] | 400 "Недопустимая роль" |

## Пути ошибок

### Ошибки базы данных

- Уникальный constraint на email нарушен (race condition при параллельной регистрации): перехват Prisma P2002, возврат 409
- Потеря соединения с PostgreSQL: 503 Service Unavailable, retry через circuit breaker

### Ошибки JWT

- Невалидная подпись токена: 401 Unauthorized
- Истекший токен: 401 с hint "TOKEN_EXPIRED" для клиента
- Отсутствие токена: 401 с hint "TOKEN_MISSING"
- Поврежденный приватный ключ RS256: приложение не стартует (fail-fast в onModuleInit)

## Безопасность

### Хранение секретов

- Пароли: bcrypt с cost factor 12 (не хранить plaintext НИКОГДА)
- Refresh-токены: SHA-256 хеш в БД (не хранить raw token в БД)
- RS256 ключи: файлы с правами 600, монтируются через Docker secrets
- OAuth секреты: переменные окружения, не коммитить в репозиторий

### Защита от атак

| Угроза | Мера |
|--------|------|
| XSS-кража токена | httpOnly + secure cookie |
| CSRF | SameSite=Strict cookie + CSRF-токен для мутаций |
| Timing attack на bcrypt | bcrypt.compare имеет константное время |
| Session fixation | Ротация refresh-токена при каждом обновлении |
| Token replay | Одноразовые refresh-токены |

### Логирование

- Успешные входы: INFO с userId, provider, IP (без пароля)
- Неудачные попытки: WARN с email, IP, причиной
- OAuth-ошибки: ERROR с provider, error code
- Никогда не логировать пароли, токены, хеши
