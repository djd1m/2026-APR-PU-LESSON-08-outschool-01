# Отчет валидации: Аутентификация и регистрация

## Оценка по INVEST

### US-P01: Регистрация и вход родителя

| Критерий | Балл (1-10) | Комментарий |
|----------|-------------|-------------|
| **I**ndependent | 9 | Полностью автономна, не зависит от других фич |
| **N**egotiable | 8 | OAuth-провайдеры можно добавлять итеративно |
| **V**aluable | 10 | Без аутентификации платформа не работает |
| **E**stimable | 8 | Стек известен (Passport, JWT, bcrypt), оценка надежна |
| **S**mall | 6 | Включает 3 стратегии (local + 2 OAuth) — на грани |
| **T**estable | 9 | Чёткие входы/выходы, Gherkin-сценарии покрывают все пути |
| **Среднее** | **8.3** | |

### US-T01: Регистрация и вход учителя

| Критерий | Балл (1-10) | Комментарий |
|----------|-------------|-------------|
| **I**ndependent | 7 | Использует тот же AuthModule, что и родитель |
| **N**egotiable | 8 | Роль TEACHER — единственное отличие |
| **V**aluable | 9 | Учителя — ключевые поставщики контента |
| **E**stimable | 9 | Минимальная дельта к US-P01 |
| **S**mall | 9 | Добавление роли — небольшое изменение |
| **T**estable | 9 | Те же тесты + проверка роли |
| **Среднее** | **8.5** | |

## Оценка по SMART

| Критерий | Балл (1-10) | Комментарий |
|----------|-------------|-------------|
| **S**pecific | 9 | Чёткие endpoints, DTO, потоки данных |
| **M**easurable | 8 | Метрики: кол-во регистраций, успешных входов, время генерации токена |
| **A**chievable | 9 | Passport + JWT — зрелый стек, все паттерны известны |
| **R**elevant | 10 | Без auth нет платформы |

| **Среднее** | **8.8** | |

## Вердикт

**READY** — Требования полные, тестируемые, хорошо декомпозированы. OAuth-стратегии можно реализовывать последовательно (сначала local, потом VK, потом Яндекс). Рекомендация: рассмотреть выделение US-P01 на 2 истории (email + OAuth) для более точной оценки.

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| VK/Яндекс API изменят формат callback | Низкая | Среднее | Абстракция OAuth в стратегиях Passport |
| RS256 ключи утекут | Очень низкая | Критичное | Docker secrets, ротация каждые 90 дней |
| Race condition при параллельной регистрации | Низкая | Низкое | Уникальный constraint + перехват P2002 |
