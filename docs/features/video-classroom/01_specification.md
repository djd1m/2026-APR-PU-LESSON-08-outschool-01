# Спецификация: Видеокласс

**Feature ID:** video-classroom
**User Stories:** US-P07, US-T04, US-C01a
**Epic:** E5
**Приоритет:** Критический (core product — живые занятия)

---

## 1. Функциональные требования

### FR-1: Создание видеокомнаты
За 15 минут до начала секции система создает Jitsi-комнату с уникальным идентификатором.

### FR-2: Доступ только по enrollment
Вход в комнату разрешен только пользователям с активным enrollment на данную секцию. JWT токен содержит enrollmentId и роль (teacher/student).

### FR-3: Функциональность комнаты
Камера, микрофон, чат, демонстрация экрана (для учителя). Запись занятия (если включена учителем).

### FR-4: Автоматическое закрытие
Через 10 минут после окончания занятия комната автоматически закрывается через BullMQ delayed job.

### FR-5: Уведомление родителя
При подключении ребенка к комнате родитель получает push-уведомление в течение 5 секунд.

## 2. Acceptance Criteria (Gherkin)

```gherkin
Feature: Видеокласс

  Scenario: Присоединение к видеозанятию
    Given до начала занятия осталось <= 15 минут
    And ребенок имеет enrollment в status=active для данной секции
    When ребенок нажимает "Присоединиться к занятию"
    Then открывается Jitsi-комната с JWT-авторизацией
    And видеозадержка < 200ms
    And родитель получает push-уведомление

  Scenario: Блокировка неавторизованного доступа
    Given пользователь не имеет enrollment на данную секцию
    When он пытается войти в комнату по прямой ссылке
    Then доступ блокируется с ошибкой "Нет доступа к занятию"
    And инцидент логируется в систему безопасности

  Scenario: Автоматическое закрытие комнаты
    Given занятие завершено (endTime прошло)
    When проходит 10 минут после окончания
    Then комната автоматически закрывается
    And все участники отключаются с сообщением "Занятие завершено"

  Scenario: Учитель начинает занятие
    Given учитель является владельцем класса
    And до начала секции <= 15 минут
    When учитель нажимает "Начать занятие"
    Then открывается комната с правами модератора
    And доступны: демонстрация экрана, управление участниками, запись

  Scenario: Переподключение при обрыве связи
    Given участник находится в комнате
    When соединение прерывается
    Then через 3 секунды начинается автоматическое переподключение
    And после восстановления участник возвращается в ту же комнату
```

## 3. Нефункциональные требования

| NFR | Требование |
|-----|-----------|
| Задержка видео | < 200ms (P95) |
| Время подключения | < 3 секунд |
| Пропускная способность | Адаптивный bitrate 150kbps-2Mbps |
| Масштабируемость | До 30 участников в комнате |
| Безопасность | JWT-авторизация, нет анонимного доступа |
| Приватность | Запись только с явного согласия учителя |

---

## 4. Ограничения

- Используется self-hosted Jitsi Meet (Docker)
- JWT подписывается HMAC-SHA256 с серверным секретом
- Комната именуется по формату `km-{sectionId}-{dateYYYYMMDD}`
- Максимальное время жизни комнаты: duration + 10 минут grace period
