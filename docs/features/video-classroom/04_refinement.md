# Уточнение: Видеокласс

## 1. Граничные случаи (Edge Cases)

### EC-1: Обрыв соединения и переподключение
**Сценарий:** Участник теряет интернет на 30 секунд.
**Решение:** Jitsi iframe API обнаруживает disconnect. Через 3 секунды начинается автоматический reconnect (встроенный в Jitsi). ConnectionStatus показывает "Переподключение...". Если > 60 секунд — предложить перезагрузить страницу.

### EC-2: Адаптация к пропускной способности
**Сценарий:** У ученика слабый интернет (< 500kbps).
**Решение:** Jitsi JVB (video bridge) автоматически снижает bitrate. Настройки: минимум 150kbps (только аудио + low-res видео), максимум 2Mbps (HD). Simulcast включен для учителя (несколько качеств одновременно).

### EC-3: Неавторизованный доступ по прямой ссылке
**Сценарий:** Кто-то получил URL комнаты и пытается войти без JWT.
**Решение:** Jitsi prosody отклоняет подключение без валидного JWT. Backend генерирует JWT только после проверки enrollment. Room name содержит UUID (не угадываемый). Инцидент логируется.

### EC-4: Комната "застряла" (не закрылась)
**Сценарий:** BullMQ job не выполнился (Redis downtime).
**Решение:** Cron-задача каждые 5 минут проверяет RoomSession со status=open и closedAt=null, где expected close time прошло. "Застрявшие" комнаты закрываются принудительно. Алерт администратору.

### EC-5: Учитель не пришел на занятие
**Сценарий:** Ученики подключились, учитель не появился в течение 10 минут.
**Решение:** После 10 минут ожидания — push-уведомление учителю "Ученики ждут вас". После 15 минут — уведомление родителям "Учитель не подключился" + пометка инцидента для администратора.

### EC-6: Одновременное подключение родителя и ребенка
**Сценарий:** Родитель открывает комнату с аккаунта, ребенок тоже хочет войти.
**Решение:** JWT привязан к enrollment, а enrollment к ребенку. Родитель входит "от имени ребенка". Второе подключение с тем же JWT — Jitsi отключает первое (стандартное поведение).

## 2. Пути ошибок

| Ошибка | HTTP | Действие frontend |
|--------|------|-------------------|
| Нет enrollment | 403 | Показать "Нет доступа", предложить запись |
| Слишком рано (> 15 мин) | 425 | Показать таймер обратного отсчета |
| Занятие завершено | 410 | Показать "Занятие завершено" |
| Jitsi недоступен | 502 | Показать "Видеосервис временно недоступен" |
| JWT generation failed | 500 | Retry, показать ошибку |

## 3. Безопасность (детская безопасность)

| Угроза | Защита |
|--------|--------|
| Посторонний в комнате | JWT + enrollment check, нет anonymous access |
| Запись без согласия | Recording только по action учителя, не автоматически |
| Неприемлемый контент в чате | Модерация: учитель может отключить чат/участника |
| Утечка JWT | Короткий TTL (duration + 10 мин), привязка к roomName |
| Буллинг между учениками | Учитель-модератор: mute, kick, disable camera |
| Перехват трафика | SRTP шифрование (встроено в Jitsi WebRTC) |

**Бонус безопасности для детей:**
- Нет прямых сообщений между учениками (только групповой чат)
- Учитель всегда модератор
- Родитель получает уведомление при входе ребенка
- Логирование всех входов/выходов из комнаты

## 4. Производительность

- Jitsi JVB: до 30 участников на одном инстансе
- При > 100 одновременных комнат: горизонтальное масштабирование JVB
- Frontend: lazy load Jitsi iframe (не грузить до нажатия "Присоединиться")
- JWT generation: < 5ms (HMAC-SHA256 — CPU-bound, быстрый)

## 5. Мониторинг Jitsi

- `jvb_conferences` (gauge) — активные конференции
- `jvb_participants` (gauge) — участники
- `jvb_packet_loss` (histogram) — потери пакетов
- Health check: GET /about/health на jitsi-web
