# Уточнение: Профили учителей

## Граничные случаи

### 1. Доступ неверифицированного учителя

| Ситуация | Поведение |
|----------|-----------|
| Профиль не верифицирован, но есть опубликованные занятия | Профиль виден в каталоге без бейджа верификации |
| Профиль не верифицирован и нет занятий | Профиль не виден в публичном списке |
| Учитель отклонен (REJECTED) | Может повторно подать на верификацию после исправлений |
| Учитель создает занятие до верификации | Допускается — модерация занятий отдельная от верификации профиля |

### 2. Манипуляция рейтингом

| Угроза | Мера защиты |
|--------|-------------|
| Самоотзывы (учитель оставляет отзыв себе) | Запрет оставлять отзыв на свои занятия (проверка authorId != teacherUserId) |
| Массовые фейковые отзывы | Отзыв может оставить только пользователь, записанный на занятие |
| Удаление негативных отзывов | Только модерация администратором, не учитель |
| Бомбинг одного учителя | Rate limit: 1 отзыв на занятие на пользователя |

### 3. Полнота профиля

| Поле | Обязательно для верификации | Обязательно для публикации |
|------|---------------------------|--------------------------|
| Имя (displayName) | Да | Да (из User) |
| Биография (bio) | Да (min 50 символов) | Нет |
| Квалификация | Да (min 20 символов) | Нет |
| Фото (avatar) | Нет | Нет |
| Предметы | Нет | Нет (но рекомендуется) |

### 4. Рейтинг с 0 отзывами

- Новый учитель: рейтинг = 0, отображается как "Нет отзывов" (не 0 звезд)
- Учитель с 1 отзывом: рейтинг отображается, но с пометкой "(1 отзыв)"
- Минимум 3 отзыва для включения в "Лучшие учителя" раздел

## Пути ошибок

### Загрузка аватара

| Ситуация | Поведение |
|----------|-----------|
| Файл > 5MB | 400 "Максимальный размер фото 5MB" |
| Недопустимый формат (gif, bmp) | 400 "Допустимые форматы: jpg, png, webp" |
| Ошибка S3/хранилища | 500, профиль сохраняется без фото, логируется ошибка |
| Битый файл (не изображение) | 400 "Файл не является изображением" (проверка magic bytes) |

### Верификация

| Ситуация | Поведение |
|----------|-----------|
| Повторный запрос в статусе PENDING | 409 "Запрос уже на рассмотрении" |
| Запрос с пустым профилем | 400 с перечислением незаполненных обязательных полей |
| Администратор approve без reason | Допускается (reason опционален для approve) |
| Администратор reject без reason | 400 "Укажите причину отклонения" |

### Конкурентные сценарии

- Два администратора одновременно обрабатывают верификацию: оптимистичная блокировка через version field
- Учитель обновляет профиль во время рассмотрения: допускается, верификация не сбрасывается

## Безопасность

- PUT /teachers/me — userId берется из JWT, нельзя обновить чужой профиль
- Биография и квалификация: санитизация HTML (strip tags), защита от XSS
- Фото: проверка content-type + magic bytes, не доверять расширению файла
- Публичные данные: только bio, qualification, subjects, rating, reviewCount (без email, userId)
- Администраторские эндпоинты: @Roles(Role.ADMIN) guard

## UX-соображения

- TeacherCard: компактное отображение (аватар 48x48, имя, предметы как теги, рейтинг звездами)
- RatingStars: полузвезды для дробных значений (4.7 = 4.5 закрашенных + 0.5 полузвезда)
- VerificationBadge: tooltip "Личность подтверждена администрацией КлассМаркет"
- Профиль учителя: занятия отображаются переиспользуемыми ClassCard компонентами
- Мобильный профиль: аватар, имя, рейтинг — sticky header при прокрутке
