# Спецификация: Расписание занятий

**Feature ID:** scheduling
**User Stories:** US-T03
**Epic:** E3
**Приоритет:** Высокий (базовая функциональность для учителей)

---

## 1. Функциональные требования

### FR-1: Создание секции
Учитель создает секцию (конкретный сеанс занятия) с указанием даты, времени начала, длительности и максимального количества учеников.

### FR-2: Обновление секции
Учитель может изменить время, длительность и лимит учеников. Изменения уведомляют записанных учеников.

### FR-3: Отмена секции
Учитель отменяет секцию. Записанные ученики получают уведомление. Платные записи переходят в процесс возврата.

### FR-4: Обнаружение конфликтов
Система проверяет пересечение новой/измененной секции с существующими секциями учителя.

### FR-5: Расписание учителя
Учитель видит недельную сетку с секциями за выбранный период.

## 2. Acceptance Criteria (Gherkin)

```gherkin
Feature: Расписание занятий

  Scenario: Создание секции без конфликтов
    Given учитель авторизован и находится на странице расписания
    When учитель заполняет форму: дата, время начала, длительность 60 мин, макс. 8 учеников
    And время не пересекается с существующими секциями
    Then секция создается со status=active
    And отображается в недельной сетке

  Scenario: Конфликт расписания
    Given учитель имеет секцию 10:00-11:00 в понедельник
    When учитель пытается создать секцию 10:30-11:30 в тот же день
    Then система показывает предупреждение "Конфликт с секцией [название]"
    And секция не создается

  Scenario: Обновление секции с записанными учениками
    Given секция имеет 3 активных записи
    When учитель изменяет время секции
    Then секция обновляется
    And все 3 записанных ученика получают email-уведомление об изменении

  Scenario: Отмена секции с записанными учениками
    Given секция имеет записи (trial + paid)
    When учитель отменяет секцию
    Then секция переходит в status=cancelled
    And trial-записи отменяются автоматически
    And paid-записи передаются в процесс возврата

  Scenario: Просмотр расписания за неделю
    Given учитель имеет секции на текущей и следующей неделе
    When учитель открывает страницу расписания
    Then отображается недельная сетка с секциями текущей недели
    And учитель может переключаться между неделями
```

## 3. Нефункциональные требования

| NFR | Требование |
|-----|-----------|
| Часовые пояса | Хранение в UTC, отображение в Europe/Moscow |
| Производительность | Загрузка расписания за неделю < 300ms |
| Валидация | Длительность: 15-180 мин, maxStudents: 1-30 |
| Конфликты | Проверка за O(log N) через индекс на startTime |

---

## 4. Ограничения

- Секция привязана к одному классу (classId)
- Учитель может создавать секции только для своих классов
- Минимальное время до секции для создания: 1 час
- Отмена секции менее чем за 24 часа — с уведомлением администратора
