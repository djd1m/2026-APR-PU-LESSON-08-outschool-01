# Уточнение: Расписание занятий

## 1. Граничные случаи (Edge Cases)

### EC-1: Переход на летнее/зимнее время (DST)
**Сценарий:** Россия не использует DST (с 2014), но некоторые пользователи могут быть в часовых поясах с DST (Европа, Америка — для экспатов).
**Решение:** Хранение строго в UTC. Конвертация через `dayjs` с библиотекой `timezone`. Frontend передает `timezone` IANA string. Все расчеты конфликтов — в UTC, что исключает DST-проблемы.

### EC-2: Секция пересекает полночь UTC
**Сценарий:** Секция 23:00-00:30 МСК → 20:00-21:30 UTC. Не пересекает полночь UTC. Но секция 01:00-02:30 МСК → 22:00-23:30 UTC предыдущего дня.
**Решение:** Алгоритм конфликтов работает с абсолютными DateTime, не с днями. Пересечение полуночи не является особым случаем.

### EC-3: Перекрытие секций при обновлении
**Сценарий:** Учитель сдвигает секцию A на время секции B.
**Решение:** При обновлении `excludeSectionId` исключает саму обновляемую секцию из проверки конфликтов. Но все остальные секции проверяются.

### EC-4: Отмена секции с записанными учениками
**Сценарий:** 5 учеников записаны (3 trial, 2 paid). Учитель отменяет секцию.
**Решение:** Trial-записи отменяются автоматически. Paid-записи передаются в очередь возвратов. Все ученики получают email-уведомление. Если отмена < 24ч до начала — уведомление администратору.

### EC-5: Изменение maxStudents ниже текущего числа записей
**Сценарий:** 6 учеников записано, учитель ставит maxStudents=4.
**Решение:** Запрет: maxStudents не может быть меньше текущего количества активных записей. Ошибка 422 с сообщением.

### EC-6: Создание секции в прошлом
**Сценарий:** Учитель указывает дату в прошлом.
**Решение:** Валидация: `startTime > NOW() + 1 hour`. Секции в прошлом создать нельзя.

## 2. Пути ошибок

| Ошибка | HTTP | Действие frontend |
|--------|------|-------------------|
| Класс не принадлежит учителю | 403 | Показать ошибку доступа |
| Конфликт расписания | 409 | Показать ConflictWarning с деталями |
| Длительность вне диапазона | 422 | Валидация на форме |
| maxStudents < enrolledCount | 422 | Показать "Нельзя уменьшить: N учеников записано" |
| startTime в прошлом | 422 | Валидация на форме + backend |
| Секция не найдена | 404 | Перенаправить на расписание |

## 3. Безопасность

| Угроза | Защита |
|--------|--------|
| Создание секции чужого класса | Проверка class.teacherId === currentUser.id |
| Изменение чужой секции | Аналогичная проверка через relation |
| Массовое создание секций | Rate limit: 50 секций/день на учителя |
| Манипуляция timezone | Whitelist IANA timezone strings |

## 4. Производительность

- Индекс `(classId, startTime)` покрывает запрос расписания и конфликты
- Запрос за неделю: WHERE classId IN (teacherClassIds) AND startTime BETWEEN range
- Кэширование не требуется: данные часто меняются, запрос быстрый по индексу
- Frontend: виртуализация WeeklyGrid при > 20 секций в неделю

## 5. Тестирование часовых поясов

Критические кейсы для тестов:
1. Europe/Moscow (UTC+3) — основной timezone
2. Asia/Vladivostok (UTC+10) — максимальный сдвиг в РФ
3. Europe/Kaliningrad (UTC+2) — минимальный сдвиг в РФ
4. Граница дня: секция 23:30 МСК → проверить корректный день в UTC
5. Новый год: секция 31 декабря 23:00 МСК → 1 января в UTC
