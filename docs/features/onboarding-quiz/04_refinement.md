# Уточнение: Онбординг-квиз

## Граничные случаи

### 1. Пропуск квиза

| Ситуация | Поведение |
|----------|-----------|
| Родитель нажимает "Пропустить" на любом шаге | Редирект в каталог без фильтров |
| При следующем входе, если children=0 | Квиз предлагается снова (banner на каталоге) |
| Родитель закрывает вкладку на шаге 2 | Данные потеряны (state в памяти), квиз начнется заново |
| Родитель обновляет страницу на шаге 3 | Данные потеряны, начало с шага 1 |

**Решение для потери данных:** сохранять промежуточное состояние в `sessionStorage`. При обновлении страницы — восстановить из sessionStorage. При закрытии вкладки — данные естественно очищаются.

### 2. Кнопка "Назад"

- На шаге 1 кнопка "Назад" скрыта (первый шаг)
- На шагах 2-4 кнопка "Назад" возвращает на предыдущий шаг
- Введенные данные сохраняются при переходе назад и вперед
- Browser back button перехватывается через `history.pushState` для каждого шага

### 3. Несколько детей

| Ситуация | Поведение |
|----------|-----------|
| Максимум детей | Лимит 5 профилей на одного родителя |
| Удаление ребенка | Soft delete, интересы сохраняются для аналитики |
| Дубликат имени | Допускается (близнецы и т.п.) |
| Разный возраст детей | Каждый профиль независим, каталог фильтруется по "активному" ребенку |

### 4. Валидация возраста

| Ввод | Результат |
|------|-----------|
| Дата в будущем | Ошибка "Дата рождения не может быть в будущем" |
| Возраст < 3 | Ошибка "Минимальный возраст 3 года" |
| Возраст > 18 | Ошибка "Максимальный возраст 18 лет" |
| Возраст ровно 3 (сегодня день рождения) | Допускается |
| Некорректный формат даты | Frontend date picker не позволяет, backend валидация IsDate |

### 5. Интересы

| Ситуация | Поведение |
|----------|-----------|
| 0 выбранных | Кнопка "Далее" неактивна + подсказка |
| 10 выбранных | Кнопка "Далее" активна, дополнительные чипы disabled |
| Более 10 | Невозможно на frontend; backend: массив max length 10 |
| Неизвестный interest ID | Backend возвращает 400 "Неизвестный интерес: xyz" |

## Пути ошибок

### Сетевые ошибки

- POST /users/children fail (500/timeout): показать сообщение "Не удалось сохранить. Попробуйте еще раз", кнопка retry
- Потеря соединения на шаге 4: retry с экспоненциальной задержкой (1s, 2s, 4s), max 3 попытки

### Конкурентные сценарии

- Два устройства одновременно проходят квиз: оба создадут профиль (допустимо, потом можно удалить лишний)
- Сессия истекла во время квиза: при POST /users/children получим 401, редирект на логин, данные квиза потеряны

## Безопасность

- Эндпоинт POST /users/children доступен только авторизованным (JwtAuthGuard)
- parentId берется из JWT, а не из body (нельзя создать ребенка другому пользователю)
- Имя ребенка: sanitize XSS (strip HTML tags), max 100 символов
- Дата рождения: серверная валидация, не доверять клиенту

## UX-соображения

- ProgressBar показывает текущий шаг (1/4) для ощущения прогресса
- На мобильных: шаги полноэкранные, кнопки "Назад"/"Далее" фиксированы снизу
- Анимация переходов между шагами (slide left/right) для ощущения непрерывности
- StepInterests: чипы расположены в grid 2 колонки на мобильном, 3-4 на десктопе
