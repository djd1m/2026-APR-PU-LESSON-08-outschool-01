# Уточнение: Панель администратора

## Граничные случаи

### Принудительное ограничение роли ADMIN
- **Проблема:** URL /admin доступен для ввода любым пользователем
- **Решение:** AdminGuard проверяет role == ADMIN + наличие 2FA; middleware на уровне Nginx дополнительно ограничивает по IP (whitelist); 403 + редирект
- **Тест:** PARENT, TEACHER → GET /admin → 403; ADMIN без 2FA → 403

### Конкурентная модерация
- **Проблема:** Два администратора одновременно модерируют одну заявку
- **Решение:** Оптимистичная блокировка через `version` поле; при конфликте — 409 с сообщением "Заявка уже обработана другим модератором"
- **Тест:** Два параллельных PATCH на одну заявку → один успех, один 409

### Кеширование статистики при изменении данных
- **Проблема:** Новые платежи поступают, но кеш аналитики устарел
- **Решение:** TTL-based кеширование (15 мин для текущего периода); прошлые периоды кешируются на 1 час; кнопка "Обновить" для принудительной инвалидации
- **Тест:** Создать платёж → через 15 мин GMV обновлён; кнопка "Обновить" → данные свежие

### Пагинация больших датасетов
- **Проблема:** 10 000+ заявок преподавателей, 50 000+ отзывов
- **Решение:** Серверная пагинация; курсорная пагинация при > 10K записей; фильтры по статусу сужают выборку
- **Тест:** 15 000 заявок, фильтр pending → пагинация по 20, каждая страница < 500ms

### Батч-операции на большом объёме
- **Проблема:** Администратор пытается одобрить 100 заявок за раз
- **Решение:** Лимит 50 элементов на батч; валидация на бэкенде; транзакция для атомарности
- **Тест:** Батч из 51 элемента → 400 "Максимум 50 элементов"

## Пути ошибок

| Ошибка | HTTP | Обработка |
|--------|------|-----------|
| Не авторизован | 401 | Редирект на /login |
| Не администратор | 403 | "Доступ только для администраторов" |
| 2FA не настроена | 403 | "Настройте двухфакторную аутентификацию" |
| Заявка уже обработана | 409 | "Заявка обработана другим модератором" |
| Батч превышает лимит | 400 | "Максимум 50 элементов за операцию" |
| Причина не указана при отклонении | 400 | "Укажите причину отклонения" |
| Некорректный период аналитики | 400 | "Период не может быть в будущем" |

## Безопасность

- **2FA обязательна:** Все эндпоинты /admin/ требуют подтверждённой 2FA (TOTP)
- **IP whitelist:** Nginx ограничивает доступ к /admin/ по IP (configurable)
- **Аудит-логирование:** Каждое действие записывается: adminId, action, target, IP, timestamp, user-agent
- **Rate limiting:** 30 req/min для админских эндпоинтов (предотвращение abuse)
- **RBAC:** Роль ADMIN назначается только через базу данных (нет UI для повышения прав)
- **Session timeout:** Админская сессия истекает через 30 минут неактивности
- **Запрет самомодерации:** Администратор не может модерировать собственные данные

## Производительность

- Метрики аналитики: materialized views для DAU/MAU, обновляются cron-задачей раз в час
- Redis-кеш: отдельный префикс `admin:` с TTL 15 мин
- Очереди модерации: индексы по `(status, created_at)` для FIFO-порядка
- Батч-операции: транзакция с batch INSERT в аудит-лог (один запрос вместо N)
- Топ-10: кешируется на 1 час (данные меняются нечасто)
