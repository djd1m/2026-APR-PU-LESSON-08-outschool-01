# Спецификация: Панель администратора

## Пользовательские истории

### US-A01: Модерация преподавателей

**Как** администратор,
**я хочу** проверять заявки преподавателей и управлять верификацией,
**чтобы** допускать на платформу только проверенных специалистов.

**Приоритет:** Critical | **Effort:** L

### US-A02: Модерация отзывов

**Как** администратор,
**я хочу** модерировать отзывы с подозрительным содержимым,
**чтобы** обеспечивать качество и безопасность контента.

**Приоритет:** High | **Effort:** M

### US-A03: Аналитика платформы

**Как** администратор,
**я хочу** видеть ключевые метрики платформы (GMV, MAU, конверсии),
**чтобы** принимать бизнес-решения на основе данных.

**Приоритет:** High | **Effort:** L

## Критерии приемки (Gherkin)

```gherkin
Feature: Вкладка "Модерация классов и преподавателей"

  Scenario: Очередь верификации преподавателей
    Given есть 5 заявок преподавателей со статусом "pending_verification"
    When администратор открывает вкладку "Преподаватели"
    Then отображается список заявок: ФИО, дата подачи, документы, видео-визитка
    And кнопки "Одобрить" и "Отклонить" для каждой заявки

  Scenario: Одобрение преподавателя
    Given администратор просматривает заявку преподавателя
    When он нажимает "Одобрить"
    Then статус преподавателя → "verified"
    And преподаватель получает email-уведомление
    And действие записывается в аудит-лог

  Scenario: Отклонение преподавателя с комментарием
    Given администратор просматривает заявку
    When он нажимает "Отклонить" и вводит причину (обязательное поле)
    Then статус → "rejected", преподаватель получает уведомление с причиной

  Scenario: Батч-операция над несколькими заявками
    Given администратор выделил 3 заявки чекбоксами
    When он нажимает "Одобрить выбранные"
    Then все 3 заявки обновляются атомарно в одной транзакции

Feature: Вкладка "Модерация отзывов"

  Scenario: Очередь отзывов на модерации
    Given есть отзывы со статусами "pending_moderation" и "flagged"
    When администратор открывает вкладку "Отзывы"
    Then отображается список: текст отзыва, рейтинг, автор, класс, флаги
    And отзывы с флагами выделены цветом

  Scenario: Публикация или отклонение отзыва
    Given администратор просматривает flagged-отзыв
    When он нажимает "Опубликовать" или "Отклонить" (с причиной)
    Then статус обновляется, рейтинг класса пересчитывается (при публикации)

Feature: Вкладка "Аналитика"

  Scenario: Отображение ключевых метрик
    Given администратор открывает вкладку "Аналитика"
    When выбран период "Месяц"
    Then отображаются: DAU/MAU, GMV, конверсия регистрация→оплата,
      средний чек, топ-10 классов, топ-10 преподавателей,
      количество активных классов/секций

  Scenario: Переключение периода
    Given отображаются метрики за месяц
    When администратор выбирает "Неделя"
    Then данные обновляются для выбранного периода

Feature: Контроль доступа

  Scenario: Не-администратор пытается открыть панель
    Given пользователь с ролью PARENT авторизован
    When он переходит на /admin
    Then возвращается 403 Forbidden и редирект на главную
```

## Нефункциональные требования

- Двухфакторная аутентификация обязательна для всех администраторов
- Все действия логируются в аудит-лог (adminId, action, target, timestamp, IP)
- Загрузка дашборда аналитики < 3 секунд (P95) с кешированием
- Батч-операции: максимум 50 элементов за раз
