# Спецификация: Каталог занятий

## Пользовательские истории

### US-P03: Поиск занятий

**Как** родитель,
**я хочу** искать занятия по ключевым словам и фильтрам,
**чтобы** найти подходящие уроки для моего ребенка.

**Приоритет:** Critical | **Effort:** L

### US-P04: Просмотр детальной информации

**Как** родитель,
**я хочу** видеть полную информацию о занятии (описание, учитель, расписание, цена),
**чтобы** принять решение о записи.

**Приоритет:** Critical | **Effort:** M

## Критерии приемки (Gherkin)

```gherkin
Feature: Поиск занятий в каталоге

  Scenario: Полнотекстовый поиск по названию
    Given в каталоге есть занятие "Математика для дошкольников"
    When родитель вводит в поиск "математика"
    Then занятие "Математика для дошкольников" отображается в результатах
    And результаты отсортированы по релевантности

  Scenario: Поиск с учетом русской морфологии
    Given в каталоге есть занятие "Уроки рисования"
    When родитель ищет "рисовать"
    Then занятие "Уроки рисования" найдено (морфологический стемминг)

  Scenario: Фильтрация по предмету
    Given в каталоге есть занятия по математике и рисованию
    When родитель выбирает фильтр предмет="Математика"
    Then отображаются только занятия по математике

  Scenario: Фильтрация по возрасту
    Given в каталоге есть занятия для возраста 5-7 и 8-12
    When родитель устанавливает фильтр ageMin=6, ageMax=10
    Then отображаются занятия с пересечением возрастного диапазона

  Scenario: Фильтрация по цене
    Given в каталоге есть занятия с ценами 500, 1000, 2000 руб
    When родитель устанавливает priceMin=800, priceMax=1500
    Then отображается только занятие за 1000 руб

  Scenario: Комбинированная фильтрация
    When родитель задает поиск "английский" + возраст 7-10 + формат "онлайн"
    Then результаты соответствуют всем трем критериям одновременно

  Scenario: Пустой результат
    When родитель ищет "квантовая физика для малышей"
    Then отображается сообщение "Ничего не найдено"
    And предлагается сбросить фильтры

  Scenario: Курсорная пагинация
    Given в каталоге 50 занятий по математике
    When родитель загружает первую страницу (limit=20)
    Then отображается 20 занятий и кнопка "Показать еще"
    When он нажимает "Показать еще"
    Then подгружаются следующие 20 занятий (cursor-based)

Feature: Детальная страница занятия

  Scenario: Просмотр полной информации
    Given занятие с id="class-123" опубликовано
    When родитель открывает /classes/class-123
    Then отображается: название, описание, учитель (с рейтингом), предмет, возраст, формат, цена, расписание
    And кнопка "Записаться" видна

  Scenario: Несуществующее занятие
    When родитель открывает /classes/nonexistent
    Then отображается страница 404
```

## Нефункциональные требования

- Время ответа поиска: < 500 мс (p95)
- Индексация: синхронизация PostgreSQL -> Elasticsearch при публикации занятия
- Русский морфологический анализатор в Elasticsearch
- Курсорная пагинация: стабильная производительность на любом offset
- SEO: детальная страница имеет meta title/description для индексации
