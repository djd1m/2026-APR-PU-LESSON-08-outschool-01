# Уточнение: Личный кабинет родителя

## Граничные случаи

### Пустые состояния
- **Проблема:** Новый родитель без детей, записей и платежей видит пустой дашборд
- **Решение:** Каждая вкладка имеет компонент EmptyState с CTA-кнопкой:
  - Расписание: "Найти классы" → каталог
  - Прогресс: "Запишите ребёнка на первый класс" → каталог
  - Платежи: "Нет платежей"
  - Дети: "Добавьте профиль ребёнка" → форма
- **Тест:** Новый аккаунт — все 4 вкладки показывают EmptyState

### Много детей (до 10)
- **Проблема:** При 10 детях расписание перегружено, прогресс не помещается
- **Решение:** В расписании — цветовая метка для каждого ребёнка; в прогрессе — dropdown-переключатель; в расписании — фильтр по ребёнку
- **Тест:** Аккаунт с 10 детьми, у каждого 3 активных курса — UI не ломается

### Пагинация истории платежей
- **Проблема:** Активный родитель с 100+ платежами — медленная загрузка
- **Решение:** Серверная пагинация по 10 элементов; курсорная пагинация при > 1000 записей
- **Тест:** 150 платежей, перелистывание 15 страниц — каждая < 300ms

### Часовой пояс в расписании
- **Проблема:** Занятие создано в UTC+3, родитель во Владивостоке (UTC+10)
- **Решение:** Все даты хранятся в UTC; frontend конвертирует через `Intl.DateTimeFormat` с автоопределением timezone; поле `timezone` передаётся на backend для серверного рендера
- **Тест:** Занятие в 10:00 MSK отображается как 17:00 для UTC+10

### Удалённый или архивированный курс
- **Проблема:** Секция архивирована после оплаты
- **Решение:** В расписании — не показывать; в прогрессе — помечать "Курс завершён"; в платежах — отображать с пометкой
- **Тест:** Архивированная секция не появляется в расписании, но видна в прогрессе

## Пути ошибок

| Ошибка | HTTP | Обработка |
|--------|------|-----------|
| Не авторизован | 401 | Редирект на /login |
| Чужой ребёнок (childId не принадлежит parentId) | 403 | "Доступ запрещён" |
| Некорректный weekStart | 400 | Валидация ISO 8601 |
| Максимум детей превышен | 400 | "Максимум 10 профилей детей" |
| Платёж не найден (чек) | 404 | "Платёж не найден" |

## Безопасность

- **Авторизация:** Все эндпоинты дашборда доступны только роли PARENT
- **Изоляция данных:** Parent guard проверяет, что запрашиваемые children принадлежат текущему пользователю
- **Персональные данные:** Дата рождения ребёнка хранится зашифрованной (AES-256); отображается только возраст
- **PDF-чеки:** Генерируются с подписью, URL с одноразовым токеном (TTL 5 минут)

## Производительность

- Агрегирующий запрос расписания: один SQL-запрос с JOIN (enrollments + sessions + classes)
- Redis-кеш прогресса: TTL 10 минут, инвалидация при обновлении attendance
- Ленивая загрузка вкладок: только активная вкладка делает API-запрос
- Prefetch: при наведении на вкладку начинается загрузка данных
