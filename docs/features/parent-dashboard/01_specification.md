# Спецификация: Личный кабинет родителя

## Пользовательские истории

### US-P08: Просмотр прогресса и управление

**Как** родитель,
**я хочу** видеть расписание, прогресс детей, историю платежей и профили детей в едином интерфейсе,
**чтобы** управлять образованием детей из одного места.

**Приоритет:** High | **Effort:** L

## Критерии приемки (Gherkin)

```gherkin
Feature: Вкладка "Расписание"

  Scenario: Отображение расписания на текущую неделю
    Given родитель авторизован и имеет детей с активными записями
    When он открывает вкладку "Расписание"
    Then отображается календарь на текущую неделю
    And каждое занятие показывает: название класса, имя ребёнка, время, преподавателя
    And занятия отсортированы по времени

  Scenario: Пустое расписание
    Given у детей родителя нет активных записей
    When он открывает вкладку "Расписание"
    Then отображается сообщение "Нет запланированных занятий"
    And кнопка "Найти классы" ведёт в каталог

  Scenario: Учёт часового пояса
    Given родитель находится в часовом поясе UTC+5 (Екатеринбург)
    When он открывает расписание
    Then время занятий отображается в его локальном часовом поясе

Feature: Вкладка "Прогресс"

  Scenario: Отображение прогресса по детям
    Given у родителя 2 ребёнка с активными курсами
    When он открывает вкладку "Прогресс"
    Then отображается переключатель между детьми
    And для каждого ребёнка: посещаемость (%), текущий уровень XP, полученные бейджи
    And список курсов с прогресс-баром (посещено N из M занятий)

  Scenario: Ребёнок без курсов
    Given у ребёнка нет ни текущих, ни завершённых курсов
    When родитель просматривает его прогресс
    Then отображается "Ваш ребёнок ещё не начал обучение"

Feature: Вкладка "Платежи"

  Scenario: История платежей с пагинацией
    Given родитель совершил 25 платежей
    When он открывает вкладку "Платежи"
    Then отображаются последние 10 платежей
    And каждый платёж: дата, сумма, класс, статус (оплачен/возврат/ожидание)
    And внизу пагинация для просмотра остальных

  Scenario: Скачивание чека
    Given платёж имеет статус "оплачен"
    When родитель нажимает иконку "Чек"
    Then скачивается PDF-чек (54-ФЗ)

Feature: Вкладка "Дети"

  Scenario: Список профилей детей
    Given у родителя 3 ребёнка
    When он открывает вкладку "Дети"
    Then отображаются карточки каждого ребёнка: имя, возраст, аватар, интересы
    And кнопка "Добавить ребёнка" и "Редактировать" для каждого

  Scenario: Добавление нового ребёнка
    Given родитель нажимает "Добавить ребёнка"
    When он заполняет: имя, дата рождения, интересы
    Then создаётся новый профиль ребёнка
    And обновляются рекомендации на главной странице
```

## Нефункциональные требования

- Время загрузки дашборда < 2 секунд (P95) включая все вкладки
- Поддержка до 10 детей на одном аккаунте
- Адаптивный дизайн: desktop, tablet, mobile
- Часовой пояс определяется автоматически через `Intl.DateTimeFormat`
