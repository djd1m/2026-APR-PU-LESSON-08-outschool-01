# Final Summary: Маркетплейс онлайн-классов для детей (Outschool RU)

## Обзор

Проект представляет собой маркетплейс живых онлайн-классов для детей от 3 до 18 лет — российский аналог платформы Outschool.com. Платформа соединяет сертифицированных преподавателей с родителями, предоставляя инструменты для поиска, бронирования, проведения и оплаты интерактивных занятий в реальном времени. Техническое решение построено на стеке Next.js + NestJS + PostgreSQL с интеграцией Jitsi для видеосвязи и ЮKassa для платежей, развёрнутое в Docker-контейнерах на российском VPS для соответствия 152-ФЗ. MVP рассчитан на запуск за 3 месяца с последующим масштабированием.

---

## Проблема и решение

### Проблема

На российском рынке отсутствует единая платформа, объединяющая независимых преподавателей и родителей для проведения живых групповых онлайн-занятий. Существующие решения либо ориентированы на репетиторство (один на один), либо представляют собой записанные курсы без интерактива. Родители тратят часы на поиск подходящих занятий в разрозненных источниках (соцсети, мессенджеры, сарафанное радио), а преподаватели лишены инструментов для привлечения учеников и управления расписанием.

### Решение

Маркетплейс с тремя ключевыми ценностными предложениями:
1. **Для родителей** — единый каталог проверенных преподавателей с отзывами, удобным поиском по возрасту/предмету/расписанию, безопасной оплатой и гарантией возврата
2. **Для преподавателей** — платформа для монетизации знаний с минимальными накладными расходами: видеосвязь, расписание, приём платежей и выплаты — всё в одном месте
3. **Для детей** — интерактивные живые занятия в малых группах (3-15 человек) с возможностью социализации

## Целевые пользователи

### Родители

| Сегмент | Описание | Потребность |
|---------|----------|-------------|
| **Активные городские родители** | 28-40 лет, доход выше среднего, мегаполисы | Развитие ребёнка вне школьной программы, удобство онлайн-формата |
| **Родители на удалёнке** | 30-45 лет, работают из дома | Занять ребёнка качественным контентом, совмещение с работой |
| **Родители в малых городах** | 25-45 лет, ограниченный выбор кружков офлайн | Доступ к преподавателям из Москвы/СПб, разнообразие предметов |

### Преподаватели

| Сегмент | Описание |
|---------|----------|
| **Профессиональные педагоги** | Учителя школ/ВУЗов, ищущие дополнительный доход |
| **Эксперты-практики** | Специалисты (художники, программисты, музыканты), желающие преподавать |
| **Начинающие репетиторы** | Студенты педвузов, набирающие опыт и портфолио |

## Технический подход

| Аспект | Решение |
|--------|---------|
| **Архитектура** | Distributed Monolith в монорепозитории |
| **Фронтенд** | Next.js 14 (App Router), TypeScript, Tailwind CSS |
| **Бэкенд** | NestJS, TypeScript, Prisma ORM |
| **База данных** | PostgreSQL 16 (основная), Redis 7 (кэш/сессии/очереди) |
| **Поиск** | Elasticsearch 8.x с русскоязычным анализатором |
| **Видео** | Jitsi Meet (self-hosted), Jibri для записи |
| **Платежи** | ЮKassa API (приём), ЮKassa Payouts (выплаты учителям) |
| **Очереди** | Bull (Redis) для фоновых задач |
| **Инфраструктура** | Docker Compose на VPS (AdminVPS/HOSTKEY, РФ) |
| **CI/CD** | GitHub Actions: lint → test → build → deploy |
| **Мониторинг** | Prometheus + Grafana + Loki (self-hosted) |

## Ключевые исследовательские выводы

| # | Вывод | Влияние на проект |
|---|-------|-------------------|
| 1 | **Outschool зарабатывает 30% комиссии** — на российском рынке с более низкой платёжеспособностью оптимальна комиссия 15-20% для старта с ростом до 25% | Установлена комиссия 20% с возможностью снижения для топ-учителей |
| 2 | **Средний размер группы на Outschool — 5-8 детей** — это оптимум для интерактива и рентабельности учителя | Минимум 3, рекомендуемый максимум 15 участников, лимит Jitsi — 30 |
| 3 | **152-ФЗ требует хранения ПД на территории РФ** — это исключает зарубежные облака для основной БД | Выбран российский VPS-хостинг, все данные в РФ |

| 5 | **Родители в РФ доверяют отзывам больше, чем сертификатам** — NPS коррелирует с количеством отзывов, а не формальными регалиями | Приоритет системы отзывов над верификацией документов в MVP |

## Таймлайн

### MVP — 3 месяца

| Месяц | Фокус | Ключевые deliverables |
|-------|-------|----------------------|
| 1 | Ядро платформы | Аутентификация, профили, каталог классов, поиск |
| 2 | Транзакции и видео | Бронирование, оплата ЮKassa, интеграция Jitsi |
| 3 | Полировка и запуск | Уведомления, отзывы, админка, деплой, нагрузочное тестирование |

### v1.0 — 6 месяцев (после MVP)

- Мобильная адаптивная версия (PWA)
- Расширенная аналитика для учителей
- Система промокодов и реферальная программа
- Интеграция с Google Calendar / Яндекс.Календарь
- AI-рекомендации классов

### v2.0 — 12 месяцев (после MVP)

- Нативные мобильные приложения (React Native)
- Подписочная модель ("безлимит" на категорию)
- Marketplace учебных материалов
- Вебинары и мастер-классы (100+ участников)
- Корпоративные программы (для школ и детских центров)
- Интеграция с ГИС "Образование"

## Риски и митигации

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| 1 | Низкое предложение учителей на старте | Высокая | Критическое | Ручной онбординг 30 учителей до запуска; 0% комиссии первые 3 месяца; контент-маркетинг в педагогических сообществах |
| 2 | Проблемы с качеством видеосвязи | Средняя | Высокое | Self-hosted Jitsi на мощном VPS; adaptive bitrate; fallback на аудио; мониторинг MOS score |
| 3 | Невыполнение требований 152-ФЗ | Низкая | Критическое | Юридическая экспертиза до запуска; хранение в РФ; шифрование ПД; аудит каждый квартал |
| 4 | Конкуренция со стороны крупных игроков (Яндекс, Сбер) | Средняя | Высокое | Фокус на нише (живые групповые занятия); лучший UX для учителей; community-driven подход |
| 5 | Мошенничество (фейковые отзывы, некачественные учителя) | Средняя | Высокое | Верификация учителей; модерация отзывов; система жалоб; автоматическое выявление аномалий |
| 6 | Масштабирование при росте нагрузки | Средняя | Среднее | Horizontal scaling VPS; CDN для статики; кэширование; мониторинг с автоалертами |
| 7 | Зависимость от ЮKassa | Низкая | Высокое | Абстракция PaymentProvider; подготовка интеграции с CloudPayments как backup |

## Ближайшие шаги

1. **Настройка инфраструктуры** — провизионирование VPS, настройка Docker, CI/CD pipeline
2. **Инициализация проекта** — монорепозиторий (Turborepo), Next.js + NestJS scaffolding, Prisma schema
3. **Ядро аутентификации** — JWT + refresh tokens, VK ID / Яндекс ID OAuth, роли и permissions
4. **Модель данных** — миграции PostgreSQL для пользователей, классов, расписания, платежей
5. **Интеграция ЮKassa** — тестовый магазин, webhook-обработка, идемпотентность
6. **Интеграция Jitsi** — self-hosted развёртывание, JWT-авторизация, кастомизация UI
7. **Каталог и поиск** — Elasticsearch-индекс, русскоязычный анализатор, фильтры
8. **MVP-релиз на staging** — полный E2E-тест с реальными учителями (бета-группа)
