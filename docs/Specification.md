# Спецификация: Маркетплейс онлайн-классов для детей (Outschool RU)

## 1. Обзор продукта

Маркетплейс живых онлайн-классов для детей 3-18 лет на российском рынке. Платформа соединяет преподавателей и родителей, предоставляя инструменты для проведения интерактивных видеозанятий, управления расписанием, оплаты и отслеживания прогресса детей.

---

## 2. Пользовательские истории (User Stories)

### 2.1. Родитель (Parent)

#### US-P01: Регистрация родителя
```
Как родитель, я хочу зарегистрироваться через VK ID, Яндекс ID или email,
чтобы быстро создать аккаунт без лишних шагов.

Acceptance Criteria:
Given родитель находится на странице регистрации
When он выбирает способ авторизации (VK ID / Яндекс ID / email+пароль)
  и заполняет обязательные поля (имя, email, телефон)
Then система создаёт аккаунт с ролью "parent",
  отправляет письмо/SMS подтверждения
  и перенаправляет на онбординг-квиз.
```

#### US-P02: Онбординг-квиз
```
Как родитель, я хочу пройти короткий квиз о возрасте и интересах ребёнка,
чтобы получить персонализированные рекомендации классов.

Acceptance Criteria:
Given родитель успешно зарегистрировался и попал на экран онбординга
When он указывает количество детей, их возраст (3-18), интересы (из предложенного списка)
  и предпочтительный формат (групповой/индивидуальный)
Then система сохраняет профили детей,
  генерирует персонализированную ленту рекомендаций
  и показывает главную страницу с подборкой классов.
```

#### US-P03: Поиск и фильтрация классов
```
Как родитель, я хочу искать классы по предмету, возрасту, цене и расписанию,
чтобы быстро найти подходящий класс для ребёнка.

Acceptance Criteria:
Given родитель находится на странице каталога классов
When он вводит поисковый запрос и/или устанавливает фильтры:
  - Возрастная группа (3-5, 6-8, 9-11, 12-14, 15-18)
  - Категория предмета (программирование, математика, английский, творчество и др.)
  - Ценовой диапазон (от-до)
  - Формат (групповой / индивидуальный / пробный)
  - Дни недели и время
  - Рейтинг преподавателя (от 4.0+)
Then система возвращает отсортированный список классов за <500ms,
  с пагинацией по 20 элементов,
  и показывает количество найденных результатов.
```

#### US-P04: Просмотр карточки класса
```
Как родитель, я хочу видеть подробную информацию о классе,
чтобы принять решение о записи.

Acceptance Criteria:
Given родитель открывает карточку класса
When страница загружается
Then отображаются: название, описание (markdown),
  фото/видео превью, профиль преподавателя (фото, рейтинг, отзывы, опыт),
  расписание ближайших секций, цена за занятие и за курс,
  возрастная группа, количество мест (занято/всего),
  отзывы родителей с рейтингом, похожие классы.
```

#### US-P05: Запись на пробное занятие
```
Как родитель, я хочу записать ребёнка на бесплатное пробное занятие,
чтобы оценить качество до покупки курса.

Acceptance Criteria:
Given родитель на карточке класса, у которого есть пробная секция
When он нажимает "Записаться на пробное",
  выбирает ребёнка из профиля и подтверждает
Then создаётся enrollment со статусом "confirmed",
  родитель получает email/push-уведомление с деталями,
  в календарь добавляется событие,
  место в секции уменьшается на 1.
```

#### US-P06: Оплата курса
```
Как родитель, я хочу оплатить курс через ЮKassa,
чтобы подтвердить запись ребёнка на все занятия.

Acceptance Criteria:
Given родитель нажал "Оплатить" на странице класса
When выбран ребёнок и секция, и родитель перенаправлен на страницу оплаты ЮKassa
Then при успешной оплате:
  - создаётся запись Payment со статусом "completed"
  - enrollment переходит в "confirmed"
  - родитель получает чек на email (54-ФЗ)
  - преподавателю начисляется сумма за вычетом комиссии
При неуспешной оплате:
  - enrollment остаётся в "pending"
  - родитель видит ошибку с возможностью повторить.
```

#### US-P07: Посещение видеозанятия
```
Как родитель, я хочу чтобы мой ребёнок мог присоединиться к видеозанятию,
чтобы участвовать в живом уроке с преподавателем.

Acceptance Criteria:
Given до начала занятия осталось <=15 минут, enrollment в статусе "active"
When ребёнок (или родитель) нажимает "Присоединиться к занятию"
Then открывается видеокомната Jitsi/LiveKit,
  с задержкой видео <200ms,
  с функциями: камера, микрофон, чат, демонстрация экрана (для преподавателя),
  с записью занятия (если включена преподавателем).
```

#### US-P08: Просмотр прогресса ребёнка
```
Как родитель, я хочу видеть прогресс ребёнка по курсам,
чтобы отслеживать его развитие и посещаемость.

Acceptance Criteria:
Given родитель открывает профиль ребёнка
When загружается страница прогресса
Then отображаются: посещённые/пропущенные занятия,
  средняя оценка от преподавателя (если есть),
  полученные бейджи и достижения,
  список текущих и завершённых курсов,
  XP и уровень в системе геймификации.
```

#### US-P09: Оставить отзыв
```
Как родитель, я хочу оставить отзыв о классе после завершения,
чтобы помочь другим родителям с выбором.

Acceptance Criteria:
Given ребёнок завершил хотя бы 1 занятие в секции
When родитель заполняет форму отзыва (рейтинг 1-5 звёзд, текст >=50 символов)
Then отзыв сохраняется со статусом "pending_moderation",
  после модерации (или автоматически через 24ч без блокирующих слов)
  публикуется и обновляет средний рейтинг класса и преподавателя.
```

#### US-P10: Реферальная программа
```
Как родитель, я хочу пригласить друзей и получить бонус,
чтобы сэкономить на следующей оплате.

Acceptance Criteria:
Given родитель находится в разделе "Пригласить друзей"
When он копирует реферальную ссылку и друг регистрируется по ней
Then после первой оплаты друга:
  - пригласивший получает бонус 500 руб. на баланс
  - приглашённый получает скидку 300 руб. на первый курс
  - максимум 20 рефералов в месяц.
```

### 2.2. Преподаватель (Teacher)

#### US-T01: Регистрация преподавателя
```
Как преподаватель, я хочу зарегистрироваться и пройти верификацию,
чтобы начать вести классы на платформе.

Acceptance Criteria:
Given пользователь выбирает роль "Преподаватель" при регистрации
When он заполняет профиль: ФИО, образование, опыт работы, специализация,
  загружает документы (диплом, справка об отсутствии судимости),
  записывает видео-визитку (1-3 минуты)
Then создаётся TeacherProfile со статусом "pending_verification",
  заявка попадает в очередь модерации,
  преподаватель получает уведомление о сроках проверки (до 48ч).
```

#### US-T02: Создание класса
```
Как преподаватель, я хочу создать карточку класса с описанием и расписанием,
чтобы привлечь учеников.

Acceptance Criteria:
Given верифицированный преподаватель находится в панели управления
When он заполняет форму создания класса:
  - Название (5-100 символов)
  - Описание (markdown, 100-5000 символов)
  - Категория и подкатегория
  - Возрастная группа (от-до)
  - Формат (разовое / курс / подписка)
  - Длительность занятия (30/45/60/90 минут)
  - Макс. учеников в группе (1-20)
  - Цена за занятие / за курс
  - Обложка (изображение, <=5MB)
  - Требования к ученикам
Then класс создаётся со статусом "draft",
  преподаватель может добавить секции (расписание),
  и отправить на модерацию (статус → "pending_review").
```

#### US-T03: Управление расписанием и записями
```
Как преподаватель, я хочу видеть расписание своих занятий и список записавшихся,
чтобы планировать нагрузку.

Acceptance Criteria:
Given преподаватель открывает раздел "Мои занятия"
When загружается страница
Then отображается: календарь занятий (неделя/месяц),
  список секций с количеством записавшихся,
  возможность отменить/перенести секцию (с уведомлением всех записавшихся),
  экспорт расписания в iCal.
```

#### US-T04: Проведение занятия
```
Как преподаватель, я хочу провести занятие в видеокомнате,
чтобы обучать учеников в реальном времени.

Acceptance Criteria:
Given до начала секции осталось <=15 минут
When преподаватель нажимает "Начать занятие"
Then открывается видеокомната с расширенными правами:
  - Управление микрофонами учеников (mute/unmute all)
  - Демонстрация экрана и whiteboard
  - Запись занятия (сохраняется в MinIO)
  - Чат с возможностью модерации
  - Список присутствующих с отметкой посещаемости.
```

#### US-T05: Просмотр заработка и вывод средств
```
Как преподаватель, я хочу видеть свой заработок и выводить средства,
чтобы получать оплату за работу.

Acceptance Criteria:
Given преподаватель открывает раздел "Финансы"
When загружается страница
Then отображаются:
  - Текущий баланс (доступно к выводу)
  - История начислений (по каждому занятию/курсу)
  - Комиссия платформы (20% от каждого платежа)
  - Минимальная сумма вывода: 1000 руб.
  - Вывод на банковскую карту через ЮKassa (срок: 1-3 рабочих дня)
  - Налоговая информация (статус самозанятого / ИП).
```

### 2.3. Ребёнок (Child)

#### US-C01: Присоединение к классу и геймификация
```
Как ребёнок, я хочу присоединяться к занятиям и получать награды,
чтобы учиться было интересно.

Acceptance Criteria:
Given ребёнок авторизован через аккаунт родителя
When он присоединяется к занятию и активно участвует
Then после завершения занятия:
  - начисляется XP (10 за посещение, бонус за серию посещений)
  - проверяется выполнение условий бейджей
  - обновляется уровень (каждые 100 XP)
  - отображается анимация получения награды.
```

#### US-C02: Просмотр достижений
```
Как ребёнок, я хочу видеть свои бейджи, уровень и прогресс,
чтобы гордиться достижениями.

Acceptance Criteria:
Given ребёнок находится в своём профиле
When он открывает раздел "Мои достижения"
Then отображаются: текущий уровень и прогресс-бар до следующего,
  коллекция полученных бейджей (с описанием и датой),
  доступные, но ещё не полученные бейджи (серые),
  статистика: общее число занятий, максимальная серия посещений.
```

### 2.4. Администратор (Admin)

#### US-A01: Модерация преподавателей
```
Как администратор, я хочу проверять заявки преподавателей,
чтобы допускать на платформу только проверенных специалистов.

Acceptance Criteria:
Given администратор открывает панель модерации
When он выбирает заявку преподавателя
Then отображаются: документы, видео-визитка, анкетные данные,
  кнопки "Одобрить" / "Отклонить" (с обязательным комментарием при отклонении),
  при одобрении — TeacherProfile.status → "verified",
  при отклонении — уведомление преподавателю с причиной.
```

#### US-A02: Модерация отзывов
```
Как администратор, я хочу модерировать отзывы,
чтобы обеспечивать качество контента на платформе.

Acceptance Criteria:
Given появился отзыв со статусом "pending_moderation" (помечен автомодератором)
When администратор просматривает отзыв
Then он может: опубликовать, отклонить (с причиной), отредактировать,
  автомодератор помечает отзывы с запрещёнными словами, слишком короткие (<50 символов),
  или подозрительные (5 звёзд от нового аккаунта) для ручной проверки.
```

#### US-A03: Аналитика и дашборд
```
Как администратор, я хочу видеть ключевые метрики платформы,
чтобы принимать бизнес-решения на основе данных.

Acceptance Criteria:
Given администратор открывает дашборд аналитики
When загружается страница
Then отображаются метрики:
  - DAU/MAU, retention (7d/30d)
  - GMV (общий объём платежей за период)
  - Конверсия: регистрация → первая оплата
  - Средний чек, LTV, CAC
  - Топ-10 классов по записям
  - Топ-10 преподавателей по рейтингу
  - Количество активных классов/секций
  с фильтрацией по периоду (день/неделя/месяц/квартал).
```

## 3. Нефункциональные требования (NFR)

### 3.1. Производительность

| Метрика | Требование | Метод измерения |
|---------|-----------|-----------------|
| Время загрузки страницы (FCP) | < 2 секунды (P95) | Lighthouse, Web Vitals |
| Время ответа API | < 300ms (P95) для CRUD, <500ms для поиска | APM (Prometheus + Grafana) |
| Задержка видеопотока | < 200ms (P95) | WebRTC stats API |
| Время полнотекстового поиска | < 500ms (P95) | Elasticsearch slow log |
| Время обработки платежа | < 5 секунд (end-to-end) | Application logs |
| Время генерации рекомендаций | < 1 секунда | MCP server metrics |
| Одновременные видеоподключения | >= 500 на один сервер Jitsi/LiveKit | Нагрузочное тестирование |

### 3.2. Безопасность

| Требование | Описание |
|-----------|----------|
| 152-ФЗ (Персональные данные) | Хранение ПД на территории РФ, согласие на обработку, право на удаление |
| 436-ФЗ (Защита детей от информации) | Возрастная маркировка контента, модерация чатов, контроль доступа |
| Шифрование at rest | AES-256 для базы данных и файлового хранилища |
| Шифрование in transit | TLS 1.3 для всех соединений, HTTPS only |
| Аутентификация | JWT (access token 15мин + refresh token 30 дней), OAuth 2.0 через VK ID / Яндекс ID |
| Авторизация | RBAC: parent, teacher, admin. Каждый эндпоинт с явной проверкой роли |
| Rate limiting | 100 req/min для авторизованных, 30 req/min для анонимных |
| OWASP Top 10 | Защита от SQL injection, XSS, CSRF, SSRF |
| Аудит логирование | Все действия администраторов, финансовые операции, изменения прав |
| Двухфакторная аутентификация | Обязательная для администраторов, опциональная для преподавателей |

### 3.3. Масштабируемость

| Параметр | Значение |
|----------|----------|
| Целевая нагрузка (M3) | 1 000 DAU, 100 одновременных видеосессий |
| Целевая нагрузка (M6) | 5 000 DAU, 500 одновременных видеосессий |
| Целевая нагрузка (M12) | 10 000 concurrent users, 2 000 видеосессий |
| Горизонтальное масштабирование API | Stateless NestJS → добавление инстансов за load balancer |
| Масштабирование БД | Read replicas (до 3), партиционирование enrollments по дате |
| Масштабирование видео | Кластер Jitsi/LiveKit (добавление JVB/SFU нод) |
| CDN | Статика, видеозаписи, изображения через CDN |

### 3.4. Доступность (Availability)

| Параметр | Значение |
|----------|----------|
| Uptime SLA | 99.9% (< 8.76 часов простоя в год) |
| RTO (Recovery Time Objective) | < 1 час |
| RPO (Recovery Point Objective) | < 15 минут (WAL-based backup) |
| Резервное копирование | Ежедневный полный бэкап PostgreSQL + инкрементальные WAL |
| Health checks | /health эндпоинт, проверки PostgreSQL, Redis, Elasticsearch |
| Мониторинг | Prometheus + Grafana + Alertmanager |

### 3.5. Доступность (Accessibility)

| Требование | Стандарт |
|-----------|---------|
| Уровень | WCAG 2.1 AA |
| Клавиатурная навигация | Полная навигация без мыши |
| Скринридеры | ARIA-атрибуты, семантический HTML |
| Контрастность | Минимум 4.5:1 для текста, 3:1 для крупного |
| Адаптивность | Поддержка экранов от 320px до 4K |
| Языки | Русский (основной), возможность добавления английского |

## 5. Ограничения и предположения

### Ограничения
1. Все серверы должны физически находиться на территории РФ (152-ФЗ)
2. Платёжная система — только ЮKassa (единственный поддерживаемый провайдер на MVP)
3. Видеосервер — self-hosted Jitsi Meet или LiveKit (без Zoom/Google Meet из-за ограничений)
4. Контент только на русском языке в MVP
5. Максимальный размер загружаемого файла: 100MB (видео), 5MB (изображения)

### Предположения
1. Преподаватели имеют стабильный интернет >= 10 Mbit/s
2. Родители используют современные браузеры (Chrome 90+, Firefox 90+, Safari 14+)
3. Средний класс содержит 5-10 учеников
4. Пиковые нагрузки — будние дни 16:00-20:00 МСК, выходные 10:00-14:00 МСК
5. Комиссия платформы 20% достаточна для операционной устойчивости

## 6. Глоссарий

| Термин | Определение |
|--------|------------|
| **Класс (Class)** | Учебная программа, созданная преподавателем. Может быть разовым или курсом. |
| **Секция (Section)** | Конкретное запланированное занятие с датой, временем и группой учеников. |
| **Запись (Enrollment)** | Факт записи ребёнка на секцию класса. |
| **Преподаватель (Teacher)** | Верифицированный пользователь, создающий и ведущий классы. |
| **Родитель (Parent)** | Пользователь, управляющий аккаунтами детей и оплачивающий классы. |
| **XP** | Очки опыта в системе геймификации, начисляемые за активность. |
| **Бейдж** | Виртуальная награда за достижение определённого условия. |
| **GMV** | Gross Merchandise Value — общий объём платежей через платформу. |
| **MCP** | Model Context Protocol — протокол интеграции AI-сервисов. |
